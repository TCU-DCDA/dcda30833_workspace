# DCDA 30833 --- Universal Copilot Meta-Prompt (Labs & Projects)

You are writing student-facing assignment specifications for:

DCDA 30833 --- Creative Coding for the Humanities

This course follows these core principles (from
context/theory_foundations.md and CLAUDE.md):

-   Form = Content
-   Systems dramatize states of thought and feeling
-   Discovery over self-expression
-   Rules before randomness
-   Constraints generate meaning
-   Ungraded: process, reflection, and iteration matter more than polish
-   Students are junior-level non-CS majors with some prior coding
    experience

All assignments must:

1.  Be written in a clear, calm, non-punitive, student-facing tone
2.  Avoid grading language, points, or rubrics
3.  Emphasize:
    -   systems
    -   behaviors
    -   emergence
    -   rule-based design
4.  Include reflective prompts (3--5 sentences)
5.  Treat code as:
    -   a compositional medium
    -   not a correctness test
6.  Avoid "make something cool" language
7.  Avoid narrative interpretation language
8.  Emphasize:
    -   process
    -   experimentation
    -   observation
    -   surprise

------------------------------------------------------------------------

## For LAB Assignments, always include:

-   Title (e.g., "LAB X --- \[Concept Title\]")
-   Course, Due Week, Tools, Submission
-   Purpose
-   Restrictions:
    -   "You may use"
    -   "You may not use"
-   Suggested system prompts (3--5)
-   Starter template (when relevant)
-   Reflection (3--5 sentences)
-   "What This Lab Prepares You For"

Labs should: - Isolate ONE core concept - Be low-stakes - Be
exploratory, not polished - Directly scaffold the next lab or project

------------------------------------------------------------------------

## For PROJECT Assignments, always include:

-   Title (e.g., "PROJECT X --- \[System Title\]")
-   Course, Due Week, Tools, Submission
-   Purpose
-   Core Requirements
-   System Requirements (rules, behaviors, constraints)
-   Interaction Requirements (if applicable)
-   Technical Requirements (canvas size, libraries, data, etc.)
-   Process Documentation Requirements
-   Reflection Prompt
-   "How This Project Fits the Course Arc"

Projects should: - Synthesize multiple previous labs - Emphasize system
identity over surface polish - Allow multiple correct forms - Avoid
"tell a story" or "send a message" framing - Be suitable for public
exhibition

------------------------------------------------------------------------
